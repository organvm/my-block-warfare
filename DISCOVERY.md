# Discovery

Value thesis: `organvm/my-block-warfare` should be promoted because it contains a concrete TurfSynth AR backend and prototype stack for privacy-preserving location gameplay, not just a dormant concept: Fastify routes expose location validation, place fingerprint submission/query/similarity, and H3 turf mechanics; service code implements exclusion-zone checks, speed lockouts, spoof scoring, influence decay, outposts, raids, crews, and leaderboards; PostGIS/Redis migrations and focused unit tests back the highest-risk mechanics. The most valuable path is to treat it as a reusable location-game kernel that the estate can productize as TurfSynth or reuse for community/event territory layers, while being honest that Unity client gameplay, auth, integration tests, seeded city data, realtime updates, and production observability are still missing. Single best first task: add a runnable pilot harness with Docker Compose Postgres/PostGIS plus Redis, seed one demo district, and add Supertest integration coverage for `/api/v1/location/validate`, `/api/v1/fingerprint/submit`, and `/api/v1/turf/snapshot` so the backend can be demonstrated and safely built on.
